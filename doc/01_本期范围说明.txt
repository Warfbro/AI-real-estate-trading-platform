文档名称：房产一站式决策与推进平台 本期范围说明
版本：V1.1（修正版）
阶段：首期 MVP
主交付端：微信小程序
辅助端：内部后台网页管理端
可选端：轻展示网页（非核心）

1. 本期目标

本期目标是完成“房产一站式决策与推进平台”的首期 MVP。
本期主交付对象是微信小程序，用户必须能够在小程序内完成核心业务闭环。
网页不作为本期主业务承载端，最多只承担内部后台或外部展示作用。

本期的核心价值不是“做一个介绍网页”，而是验证以下事项：
1）用户是否愿意在微信小程序中完成需求表达、房源导入、房源比较、风险确认和推进下一步。
2）系统是否能显著降低多平台查找、比较和确认风险的成本。
3）系统是否能把用户从“看一看”推进到“发起咨询/预约/复核”。

2. 本期主交付边界

2.1 主交付
本期主交付为微信小程序 MVP。
以下核心能力必须优先在小程序内实现：
1）需求录入
2）房源导入
3）候选结果
4）房源比较
5）风险确认
6）推进下一步
7）用户历史记录

2.2 辅助交付
本期允许有两个辅助端，但都不是主验收对象：
A. 内部后台网页管理端
用于顾问和运营查看线索、更新状态、查看日志。
B. 轻展示网页
仅用于项目介绍、亮点展示、演示截图和扫码引导进入小程序。
如果资源不足，可延期。

3. 本期必须完成的模块

3.1 微信小程序用户端（P0）
1）首页/进入页
2）登录与基础授权
3）买房需求录入
4）需求摘要确认
5）房源导入（链接、文本、截图中至少支持两种）
6）候选房源列表
7）房源详情
8）2~5 套房源比较
9）风险确认页
10）下一步动作中心
11）我的需求/我的房源/我的比较/我的报告/我的预约咨询

3.2 后台管理端（P1）
1）线索列表
2）线索详情
3）顾问状态更新
4）预约/咨询承接
5）日志查看

3.3 轻展示网页（P2，可选）
1）项目简介
2）功能亮点
3）演示内容
4）扫码进入小程序

4. 本期明确不做

1）不做网页端核心业务闭环
2）不做原生 App
3）不做完整房产门户网站
4）不做全网自动抓取所有平台
5）不做复杂地图找房
6）不做复杂 IM 即时聊天
7）不做自动签约、自动过户、自动贷款审批
8）不做多租户机构 SaaS
9）不做复杂 BI 系统
10）不做全量 CRM

5. 本期主流程

小程序首页
-> 登录/授权（必要时）
-> 输入买房需求
-> 生成需求摘要
-> 导入房源
-> 生成候选房源
-> 选择 2~5 套房源比较
-> 查看比较结果
-> 查看风险确认
-> 点击下一步
-> 发起咨询 / 预约 / 报告发送 / 人工复核 / 稍后继续

6. 验收口径

本期是否完成，以微信小程序主流程是否跑通为准，不以展示网页是否完成为准。
必须满足：
1）用户能在小程序里提交需求
2）用户能在小程序里导入房源
3）用户能在小程序里完成比较
4）用户能在小程序里看到风险确认结果
5）用户能在小程序里推进下一步
6）后台能承接该动作
7）关键数据可落库、可追溯

7. 优先级

P0：微信小程序核心闭环
P1：后台管理端最小能力
P2：展示网页

结论：
首期 MVP 的本质是“微信小程序上的最小可用业务闭环”，不是“一个展示网页项目”。
