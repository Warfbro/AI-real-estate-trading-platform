文档名称：房产一站式决策与推进平台 功能清单 / 简版 PRD
版本：V1.1（修正版）
适用阶段：首期微信小程序 MVP

1. 产品定位

本产品是一个以微信小程序为主承载端的买房决策与推进工具。
核心价值：
1）帮用户更快表达清楚需求
2）帮用户把多平台房源统一标准化
3）帮用户完成高质量比较
4）帮用户更早看见风险
5）帮用户低门槛进入下一步

2. 用户角色

A. 买房用户
目标：从信息分散、难比较、难确认，转为快速筛选、比较、确认并推进下一步。

B. 辅助决策者
目标：查看比较结论和风险摘要，辅助判断。

C. 顾问/经纪支持人员
目标：承接咨询、预约、复核请求并推进状态。

D. 运营/管理员
目标：维护后台、查看日志、处理异常。

3. 小程序端功能模块

3.1 首页
谁用：买房用户
输入：无
输出：进入需求录入、导入房源、查看历史记录
主要功能：
1）展示产品价值主张
2）进入需求录入
3）进入房源导入
4）继续上次进度
5）查看我的空间

3.2 登录/授权
谁用：买房用户
输入：微信授权信息、补充手机号/邮箱（如需要）
输出：用户身份、用户档案
说明：
未登录用户可浏览部分页面，但收藏、保存、咨询、预约、发送报告前必须登录。

3.3 需求录入
谁用：买房用户
输入：
1）自然语言需求
2）结构化补充项：预算、区域、用途、优先条件、不能接受项、最大顾虑
输出：
1）需求摘要
2）结构化需求对象
3）可进入候选页
要求：
支持修改并重新生成摘要。

3.4 房源导入
谁用：买房用户
输入：
1）房源链接
2）房源文本
3）房源截图（至少支持两种，目标支持三种）
输出：
1）标准化房源对象
2）原始内容保存
3）导入结果提示
要求：
如识别失败，允许手动补全关键字段。

3.5 候选房源列表
谁用：买房用户
输入：需求对象、筛选条件
输出：候选房源列表、推荐理由
支持：
1）预算/面积/户型/区域/楼龄/电梯等筛选
2）收藏
3）加入比较
4）查看详情

3.6 房源详情
谁用：买房用户
输入：房源 ID
输出：完整房源信息、匹配说明、缺失字段提示
支持：
1）查看原始导入内容
2）查看结构化结果
3）查看适配说明
4）加入比较
5）进入风险确认

3.7 房源比较
谁用：买房用户
输入：2~5 套房源 + 当前需求
输出：
1）对比矩阵
2）核心差异
3）优缺点
4）推荐排序
5）继续确认问题清单
支持：
1）保存比较
2）生成报告
3）进入风险确认
4）推进下一步

3.8 风险确认
谁用：买房用户
输入：单套或多套房源 + 需求对象
输出：
1）总体风险等级
2）风险标签
3）风险解释
4）人工复核建议
5）推荐下一步动作
说明：
首期以规则打标为主，AI 负责解释。

3.9 下一步动作中心
谁用：买房用户
输入：比较或风险结果
输出：动作记录和状态
支持动作：
1）咨询顾问
2）预约带看
3）发送比较报告
4）请求人工复核
5）稍后继续

3.10 我的空间
谁用：买房用户
输出：
1）我的需求
2）我的房源
3）我的比较
4）我的报告
5）我的预约/咨询
6）最近继续入口

4. 后台管理端功能模块

4.1 线索列表
谁用：顾问/运营
输出：线索摘要、状态、更新时间

4.2 线索详情
谁用：顾问/运营
输出：
1）用户需求
2）导入房源
3）比较结果
4）风险确认
5）动作记录
6）日志

4.3 状态更新
谁用：顾问/运营
支持状态：
未跟进、跟进中、待确认、已预约、已完成、已关闭

5. 功能优先级

P0（首期必须）
首页、登录、需求录入、房源导入、候选页、详情页、比较页、风险确认页、下一步动作中心、我的空间、后台线索列表和详情、状态更新

P1（首期可做但可简化）
报告发送、人工复核队列、埋点补全、更多筛选项

P2（后续）
展示网页、更多平台接入、地图与通勤、更复杂规则

6. 输入输出总览

需求录入：
输入：自然语言 + 结构化条件
输出：需求摘要 + 结构化需求对象

房源导入：
输入：链接/文本/截图
输出：标准化房源对象 + 原始内容

房源比较：
输入：2~5 套房源 + 当前需求
输出：对比矩阵 + 推荐结论 + 继续确认清单

风险确认：
输入：房源对象 + 规则结果
输出：风险等级 + 风险解释 + 建议动作

下一步动作：
输入：用户动作
输出：动作记录 + 后台承接状态

7. 成功标准

如果用户能够在小程序内完成：
需求提交 -> 房源导入 -> 比较 -> 风险确认 -> 咨询/预约/复核
则说明首期 PRD 闭环成立。
