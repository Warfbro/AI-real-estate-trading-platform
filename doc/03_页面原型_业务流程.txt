文档名称：页面原型 + 业务流程
版本：V1.1（修正版）
适用阶段：首期微信小程序 MVP

2）登录/授权页（必要时弹出）
3）需求录入页
4）候选房源列表页
5）房源导入页
6）房源详情页
7）房源比较页
8）风险确认页
9）下一步动作页/弹层
10）我的空间页
11）我的需求列表页
12）我的房源列表页
13）我的比较列表页
14）我的报告列表页
15）我的预约/咨询列表页

B. 后台管理端网页
1）线索列表页
2）线索详情页
3）状态更新页/弹层
4）日志查看页

C. 可选轻展示网页
1）项目介绍页
2）演示截图页
3）扫码引导页
说明：不属于首期主流程。

2. 小程序端关键页面原型说明

2.1 首页
核心区域：
1）品牌名与价值主张
2）“开始选房”主按钮
3）“导入房源”次按钮
4）“继续上次进度”入口
5）“我的”入口
页面跳转：
首页 -> 需求录入页
首页 -> 房源导入页
首页 -> 我的空间页

2.2 需求录入页
区域：
1）自然语言输入框
2）结构化条件卡片
3）系统理解摘要
4）重新生成按钮
5）保存并继续按钮
跳转：
需求录入页 -> 候选房源列表页
需求录入页 -> 房源导入页（如用户先导入房源）

2.3 房源导入页
区域：
1）链接输入
2）文本粘贴
3）截图上传
4）导入结果列表
5）失败重试/手动补全
跳转：
房源导入页 -> 候选房源列表页
房源导入页 -> 房源详情页

2.4 候选房源列表页
区域：
1）当前需求摘要
2）筛选器
3）房源列表
4）推荐理由
5）收藏按钮
6）加入比较按钮
7）查看详情按钮
跳转：
候选页 -> 房源详情页
候选页 -> 房源比较页
候选页 -> 风险确认页（单房源）
候选页 -> 下一步动作页（少用）

2.5 房源详情页
区域：
1）基础字段
2）来源信息
3）原始导入内容
4）结构化结果
5）适配说明
6）缺失字段提示
7）加入比较按钮
8）进入风险确认按钮
跳转：
详情页 -> 房源比较页
详情页 -> 风险确认页

2.6 房源比较页
区域：
1）对比矩阵
2）差异高亮
3）推荐结论
4）继续确认清单
5）保存比较
6）生成报告
7）进入风险确认
8）下一步按钮
跳转：
比较页 -> 风险确认页
比较页 -> 下一步动作页
比较页 -> 报告发送

2.7 风险确认页
区域：
1）总体风险等级
2）风险清单
3）风险解释
4）人工复核提示
5）推荐动作按钮
跳转：
风险确认页 -> 下一步动作页

2.8 下一步动作页/弹层
动作：
1）咨询顾问
2）预约带看
3）发送报告
4）请求人工复核
5）稍后继续
动作结果：
提交成功 -> 成功提示 -> 返回我的空间或停留当前页

2.9 我的空间
区域：
1）最近继续
2）我的需求
3）我的房源
4）我的比较
5）我的报告
6）我的预约/咨询
跳转：
我的空间 -> 各列表页
各列表页 -> 对应详情/继续流程页

3. 后台管理端页面原型说明

3.1 线索列表页
字段：
用户、需求摘要、动作类型、当前状态、最后更新时间
操作：
查看详情、筛选状态、搜索

3.2 线索详情页
区域：
1）用户信息
2）需求对象
3）房源对象
4）比较结果
5）风险确认
6）下一步动作
7）日志
8）状态更新

4. 小程序主业务流程

流程 A：标准买房流程
首页
-> 需求录入页
-> 候选房源列表页
-> 房源详情/比较页
-> 风险确认页
-> 下一步动作页
-> 咨询/预约/复核成功

流程 B：用户先有房源流程
首页
-> 房源导入页
-> 房源详情页
-> 需求录入页
-> 候选房源列表页
-> 比较页
-> 风险确认页
-> 下一步动作页

流程 C：继续上次进度
首页
-> 继续上次进度
-> 上次停留页
-> 完成当前动作

5. 状态与页面联动原则

1）未登录可浏览首页、部分候选结果，但不能保存和推进。
2）已登录才可保存需求、房源、比较、报告和动作记录。
3）当比较对象不足 2 套时，不允许进入正式比较结论态。
4）当高风险命中时，风险页必须突出人工复核动作。
5）任何动作提交后都必须有结果反馈。

6. 原型交付建议

小程序优先画低保真：
首页、需求录入、导入、候选、详情、比较、风险、我的
后台画最小低保真：
线索列表、线索详情、状态更新
展示网页可不画或只画 1 页示意。
