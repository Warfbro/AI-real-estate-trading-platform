2. 小程序端关键页面原型说明

2.1 首页
核心区域：
1）品牌名与价值主张
2）“开始选房”主按钮
3）“导入房源”次按钮
4）“继续上次进度”入口
5）“我的”入口
页面跳转：
首页 -> 需求录入页
首页 -> 房源导入页
首页 -> 我的空间页

2.2 需求录入页
区域：
1）自然语言输入框
2）结构化条件卡片
3）系统理解摘要
4）重新生成按钮
5）保存并继续按钮
跳转：
需求录入页 -> 候选房源列表页
需求录入页 -> 房源导入页（如用户先导入房源）

2.3 房源导入页
区域：
1）链接输入
2）文本粘贴
3）截图上传
4）导入结果列表
5）失败重试/手动补全
跳转：
房源导入页 -> 候选房源列表页
房源导入页 -> 房源详情页

2.4 候选房源列表页
区域：
1）当前需求摘要
2）筛选器
3）房源列表
4）推荐理由
5）收藏按钮
6）加入比较按钮
7）查看详情按钮
跳转：
候选页 -> 房源详情页
候选页 -> 房源比较页
候选页 -> 风险确认页（单房源）
候选页 -> 下一步动作页（少用）

2.5 房源详情页
区域：
1）基础字段
2）来源信息
3）原始导入内容
4）结构化结果
5）适配说明
6）缺失字段提示
7）加入比较按钮
8）进入风险确认按钮
跳转：
详情页 -> 房源比较页
详情页 -> 风险确认页

2.6 房源比较页
区域：
1）对比矩阵
2）差异高亮
3）推荐结论
4）继续确认清单
5）保存比较
6）生成报告
7）进入风险确认
8）下一步按钮
跳转：
比较页 -> 风险确认页
比较页 -> 下一步动作页
比较页 -> 报告发送

2.7 风险确认页
区域：
1）总体风险等级
2）风险清单
3）风险解释
4）人工复核提示
5）推荐动作按钮
跳转：
风险确认页 -> 下一步动作页

2.8 下一步动作页/弹层
动作：
1）咨询顾问
2）预约带看
3）发送报告
4）请求人工复核
5）稍后继续
动作结果：
提交成功 -> 成功提示 -> 返回我的空间或停留当前页

2.9 我的空间
区域：
1）最近继续
2）我的需求
3）我的房源
4）我的比较
5）我的报告
6）我的预约/咨询
跳转：
我的空间 -> 各列表页
各列表页 -> 对应详情/继续流程页

3. 后台管理端页面原型说明

3.1 线索列表页
字段：
用户、需求摘要、动作类型、当前状态、最后更新时间
操作：
查看详情、筛选状态、搜索

3.2 线索详情页
区域：
1）用户信息
2）需求对象
3）房源对象
4）比较结果
5）风险确认
6）下一步动作
7）日志
8）状态更新

4. 小程序主业务流程

流程 A：标准买房流程
首页
-> 需求录入页
-> 候选房源列表页
-> 房源详情/比较页
-> 风险确认页
-> 下一步动作页
-> 咨询/预约/复核成功

流程 B：用户先有房源流程
首页
-> 房源导入页
-> 房源详情页
-> 需求录入页
-> 候选房源列表页
-> 比较页
-> 风险确认页
-> 下一步动作页

流程 C：继续上次进度
首页
-> 继续上次进度
-> 上次停留页
-> 完成当前动作

5. 状态与页面联动原则

1）未登录可浏览首页、部分候选结果，但不能保存和推进。
2）已登录才可保存需求、房源、比较、报告和动作记录。
3）当比较对象不足 2 套时，不允许进入正式比较结论态。
4）当高风险命中时，风险页必须突出人工复核动作。
5）任何动作提交后都必须有结果反馈。

6. 原型交付建议

小程序优先画低保真：
首页、需求录入、导入、候选、详情、比较、风险、我的
后台画最小低保真：
线索列表、线索详情、状态更新
展示网页可不画或只画 1 页示意。

7. 2026-03-13 页面与流程调整补充
1）页面结构新增：AI 咨询页（pages/ai/index）。
2）首页跳转关系调整为：
首页 -> AI 咨询页
首页 -> 传统搜索页
首页 -> 我的空间页
3）AI 咨询页作为 AI 主承接页：
无需求 -> 需求录入页
有需求无房源 -> 房源导入页
已勾选 2 套及以上 -> 房源比较页
未满足比较条件 -> 传统搜索页
4）需求录入页、房源导入页成功后回流 AI 咨询页，不再直接推进到传统搜索页。
5）候选房源列表页调整为传统搜索页：
允许无需求独立进入；
显示 AI 偏好参考但不作为硬阻断；
主承接动作改为“交给 AI 分析”。
6）房源详情页调整为传统搜索详情页：
保留加入已选、返回搜索；
新增“交给 AI 分析”；
不再作为直接进入风险确认和比较页的主路径。
7）标准流程更新为：
首页 -> AI 咨询页 -> 需求录入 / 房源导入 / 传统搜索选房 -> 房源比较页 -> 风险确认页 -> 下一步动作页。
8）独立搜索流程更新为：
首页 -> 传统搜索页 -> 房源详情页 / 勾选房源 -> AI 咨询页 -> 房源比较页及后续流程。

=== 2026-03 视觉与交互设计方针统一说明 ===

1. 整体基调：
- 风格：温暖、专业、可信、家居化（木质/奶油色调）。
- 交互主题：“缓”。买房是一个缓慢挑选、从容决策的过程，动效设计和交互反馈需保持平滑、柔和（如页面缓缓展开、面板平滑升起），避免急躁的视觉突变打断用户思考。
- 目的：降低用户对高客单价房产交易的防备感，传达安心与温度。

2. 核心色彩：
- 主背景色：#FAF8F5（米白/奶油色），用于替代大面积冷白色。
- 主要强调色/按钮：#5C4033（深胡桃木/咖啡色），用于主按钮、TabBar常态。
- 次级点缀色/高亮：#E4D4C8（浅原木色/奶棕色），用于选中状态、标签背景。
- 文本颜色：#333333（主文本），#666666（次文本）。

3. 布局与组件约束：
- 导航架构：采用全局底部自定义 TabBar，包含“首页（左）、AI主入口（中，悬浮凸起）、我的（右）”。
- 卡片设计 (Card)：全站内容白底卡片结构，圆角统一 24rpx，阴影带棕色调（rgba(92, 64, 51, 0.08)）。
- 安全区：所有信息流页底部保留 180rpx 以上 padding，配合 env(safe-area-inset-bottom) 防遮挡。

4. 首页结构改版：
- 取消旧版横幅广告与冗余进度卡。
- 引入顶部搜索、横向滑动“热门房源”与垂直流式“猜你喜欢”列表，数据均依托现有房源数据库，不新增资讯模型。
