文档名称：核心数据表 / 字段字典
版本：V1.1（修正版）
适用阶段：首期微信小程序 MVP

1. 设计原则

1）围绕“微信小程序用户主流程”建模
2）原始内容与结构化结果同时保存
3）比较结果、风险结果、动作记录必须落库
4）支持小程序端继续上次进度
5）支持后台承接线索

2. 核心对象总览

1）users 用户
2）user_profiles 用户档案
3）buyer_intakes 买房需求
4）chat_sessions AI 会话
5）chat_messages AI 消息归档
6）listing_import_jobs 房源导入任务
7）listings 房源对象
8）listing_versions 房源版本
9）favorites 收藏
10）comparison_reports 比较报告
11）risk_checks 风险确认
12）next_actions 下一步动作
13）appointments 预约记录
14）advisor_leads 顾问线索
15）activity_logs 操作日志

6. 字段命名原则

1）统一使用英文 snake_case
2）所有对象都要有 created_at / updated_at
3）结构化结果优先放 JSON 字段，便于首期快速迭代
4）展示字段与计算字段分离
5）所有状态字段必须在状态流转文档中有定义

=== 2026-03 视觉与交互设计方针统一说明 ===

1. 整体基调：
- 风格：清透、专业、可信，以首页现有蓝灰到米白渐变为全站统一视觉方向。
- 交互主题：“缓”。买房是一个缓慢挑选、从容决策的过程，动效设计和交互反馈需保持平滑、柔和（如页面缓缓展开、面板平滑升起），避免急躁的视觉突变打断用户思考。
- 目的：降低高客单价交易的压迫感，强化“信息清晰、决策从容”的体验。

2. 核心色彩：
- 主背景层：低饱和蓝灰到米白渐变（#D6E0EF -> #DDE6F2 -> #F8F3EA），用于全站页面背景。
- 卡片/容器：白色或近白（#FFFFFF / rgba(255,255,255,0.96)），配合蓝灰描边。
- 主要强调色/按钮：橙色系（#F06A23 / #FF651C），用于主按钮、价格高亮、关键 CTA。
- 次级点缀色/标签：蓝灰色（#D6E0EF），用于标签、筛选、弱按钮和补充信息块。
- 文本颜色：#243247（主文本），#6B7A8F（次文本）。

3. 布局与组件约束：
- 导航架构：采用全局底部自定义 TabBar，包含“首页（左）、AI主入口（中，悬浮凸起）、我的（右）”。
- 卡片设计 (Card)：全站内容统一白卡 + 蓝灰描边结构，圆角统一 24rpx，阴影偏冷灰蓝（如 rgba(102, 123, 156, 0.12)）。
- 安全区：所有信息流页底部保留 180rpx 以上 padding，配合 env(safe-area-inset-bottom) 防遮挡。

4. 页面风格对齐要求：
- 首页作为全站视觉基准；其蓝灰渐变背景、白卡容器、橙色强调和轻量阴影应向登录、录入、导入、比较、风险、我的等页面统一。
- 不再新增木质/奶油色主视觉页面；如局部已有暖色资产，仅允许作为极小面积辅助色，不得主导页面观感。

7. 2026-03-24 补充：decision_sessions（AI 最小决策闭环）

对象：decision_sessions
用途：承接 AI 页结构化决策闭环的最小状态，不替代 chat_sessions/chat_messages。

字段：
- decision_session_id：主键
- user_id：所属用户
- chat_session_id：关联 chat_sessions，可空
- intake_id：关联 buyer_intakes，可空
- status：active/completed/archived
- current_stage：clarifying/ranking/pairwise/critique/relaxation
- selected_listing_ids：用户当前带入决策的房源 ID 数组
- candidate_listing_ids：当前候选池房源 ID 数组
- state_json：结构化决策状态，当前仅保留 hard_constraints / soft_weights / pairwise_memory / critique_memory / blockers
- result_json：当前结果快照，当前仅保留 buckets / top_listing_ids / relaxation_options
- created_at：创建时间
- updated_at：更新时间

说明：
1）本期只新增 decision_sessions 1 个集合，不新增 preference_states。
2）pairwise、critique、relax 的中间状态统一存入 state_json，保证 Sprint 1 可运行、可回退。
3）三档候选 bucket 固定为 stable / balanced / value。
4）该集合由云函数 decisionEngine 读写，activity_logs 仅保留关键动作留痕，不承担主状态存储。
