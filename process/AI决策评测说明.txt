文档名称：AI 决策离线评测说明
版本：V1.0
日期：2026-03-25
适用范围：`decisionEngine` 最小结构化决策闭环的离线黄金集与 baseline 对比

一、目标
本说明用于补齐 AI 决策模块的最小离线评测闭环，不改动小程序线上主流程。
本轮只做 3 件事：
1）沉淀一份可解释、可手查的黄金测试数据集；
2）固定当前方案与 3 个 baseline 的离线评测口径；
3）用现有 Node 脚本直接跑出可复现结果。

二、文件
1）数据集：`process/ai_decision_eval_dataset.json`
2）评测脚本：`process/ai_decision_baseline_eval.js`

三、数据集边界
本轮数据集只覆盖当前仓库已稳定支持的字段与状态：
- `city`
- `district`
- `price_total`
- `area_sqm`
- `layout_desc`
- `elevator_flag`
- `missing_fields_json`
- `pairwise`
- `critique`

不额外假设本期尚未正式进入决策内核的学区、通勤、家庭协同、完整贝叶斯后验等能力。

四、当前方案与 baseline 定义
1）Current System
- 直接调用 `cloudfunctions/decisionEngine`
- 完整走 `start -> pairwise/critique -> state`
- 使用 `decision_sessions.state_json` 中的显式偏好状态更新

2）Baseline 1：纯规则筛选排序
- 复用当前硬约束过滤
- 不使用 pairwise / critique
- 使用简化的规则排序：`layout_match -> price asc -> area desc -> elevator -> missing_fields`

3）Baseline 2：LLM 直出推荐
- 直接调用 `openai_compatible` provider
- 把需求、后续反馈和候选房源文本化后一次性交给模型
- 不使用结构化 `decision_sessions` 状态

4）Baseline 3：去掉显式偏好状态的简化版
- 复用 `createInitialDecisionState + filterFeasibleListings + rankListings`
- 只用初始上下文，不执行 pairwise / critique 更新
- 等价于“保留当前排序器，但不保留显式偏好状态”

五、指标
脚本默认输出以下指标：
1）`top1_accuracy`
2）`hit_at_3`
3）`ndcg_at_3`
4）`top1_rule_hit_rate`
5）`pairwise_win_rate`

说明：
- `top1_rule_hit_rate` 用于看第一推荐是否满足黄金约束；
- `pairwise_win_rate` 只在存在 pairwise 反馈的 case 上统计；
- 当前数据集规模是“最小可解释集”，优先用于回归，不追求统计显著性。

六、运行方式
1）基础运行
`node process/ai_decision_baseline_eval.js`

2）输出 JSON 报告
`node process/ai_decision_baseline_eval.js --out process/ai_decision_eval_report.json`

七、环境约束
1）脚本依赖本机 `node`
2）Baseline 2 需要可用的 OpenAI 兼容 provider 凭证
3）若未显式配置环境变量，脚本会按现有 smoke 约定尝试读取：
- `OPENAI_COMPAT_API_KEY`
- `BAILIAN_API_KEY`
- `DASHSCOPE_API_KEY`
- 本地 `新需求/apikey`
4）为避免离线评测被云函数默认 12 秒窗口误伤，脚本在本地运行时默认把 `OPENAI_COMPAT_TIMEOUT_MS` 提升到 `30000ms`；不改动线上云函数默认值

八、回退方式
本轮新增均为 `process/` 目录内的测试资产。
如需回退，直接按提交边界移除：
- `process/ai_decision_eval_dataset.json`
- `process/ai_decision_baseline_eval.js`
- `process/AI决策评测说明.txt`

九、定位
这份离线评测不是新的产品能力，而是当前交付态下对 AI 决策闭环补一层“可回归、可比较、可持续迭代”的验证基础。
