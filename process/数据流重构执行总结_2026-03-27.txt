【数据流转变】

改造前（直接操作）：
页面 -> storage.set(key, value) -> storage 中

改造后（三层缓存 + 版本控制）：
页面 -> repo.operation() 
  -> repo 内存缓存更新
  -> storage.setWithVersion() 
  -> 异步云端同步
  -> 返回结果 {status, data, version, updated_at}

【版本控制规则】

基本规则：
- 所有核心对象都补充 version 字段（初始 "1"）
- 每次更新都递增 version（"1" -> "2" -> "3"...）
- 同步 updated_at（ISO 8601 格式）- 所有对象都补充 user_id 字段（便于多用户隔离）

冲突解决：
- 简单字段：按 updated_at 最新覆盖
- 复杂对象：按 version 比较，本地 < 云端时覆盖
- 消息记录：按 message_id 去重追加（不做覆盖）
