文档名称：格式.txt
版本：V1.0
用途：用于规范process.txt。
原则：每次输出都必须同时体现“整个开发过程到哪一步了”和“这一次执行到哪了”。

固定输出格式如下：

【开发总过程规划】
开发总目标：{一句话描述整个开发最终目标}
开发总过程：{按顺序列出完整开发过程，例如：项目初始化 -> 基础能力搭建 -> 数据结构与接口骨架开发 -> 核心页面开发 -> 核心业务逻辑开发 -> 前后端联调 -> 测试与修复 -> 演示与交付收口}
当前所处步骤：{当前位于开发总过程中的哪一步}
当前步骤目标：{当前步骤要达成什么}
当前步骤完成标准：{达到什么程度才算这一步完成}

【当前执行进度】
当前开发项：{当前正在开发的模块/页面/接口/功能}
当前状态：{未开始 / 开发中 / 联调中 / 已完成}
已完成：{本次执行后已经完成的内容}
进行中：{当前仍在处理的内容}
下一步：{紧接着要处理的内容}

输出要求：
1. 必须严格按照上述两个部分输出。
2. “开发总过程规划”表示整个开发链路，不是本次局部任务。
3. “当前执行进度”表示本次执行后的真实状态。
4. “开发总过程”必须写完整链路，不能只写当前一步。
5. “当前所处步骤”必须能在“开发总过程”中对应上。
6. 内容必须与真实开发状态一致，不允许虚报完成。
7. 没有内容时写“无”。
8. 不允许输出与开发过程无关的信息。
9. 如果当前开发项已完成，则“进行中”写“无”，并把“下一步”切换为下一个开发项。
10. 如果当前步骤已完成，则同步更新“当前所处步骤”“当前步骤目标”“当前步骤完成标准”。

示例：

【开发总过程规划】
开发总目标：完成“会选房”微信小程序 MVP 的可运行开发版本，并能进行联调、测试和演示
开发总过程：项目初始化 -> 基础能力搭建 -> 数据结构与接口骨架开发 -> 核心页面开发 -> 核心业务逻辑开发 -> 前后端联调 -> 测试与修复 -> 演示与交付收口
当前所处步骤：核心页面开发
当前步骤目标：完成房源列表页、筛选面板、详情页等核心页面的开发与基础交互
当前步骤完成标准：核心页面可进入、可退出、可展示、可触发基本操作

【当前执行进度】
当前开发项：房源列表页与筛选面板
当前状态：开发中
已完成：已完成房源列表页基础结构和筛选面板静态布局
进行中：正在接入筛选条件变化后的列表刷新逻辑
下一步：联调房源筛选请求与结果回显
